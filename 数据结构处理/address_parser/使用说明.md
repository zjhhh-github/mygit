## 模块简介

`address_parser` 是一个**中文地址结构化解析库**，输入一段包含姓名、手机号、地址的混合文本，自动提取并返回：

- 收件人姓名
- 手机号
- 省份 / 城市 / 区县
- 详细地址（去掉省市区后的剩余部分）

支持：省份简称（如"内蒙古"→"内蒙古自治区"）、城市/区县简称（如"海淀"→"海淀区"）、常见错别字纠错（如"郑洲"→"郑州市"）、ES 模糊搜索、LLM 智能兜底等多级解析策略。

## 文件结构

```
address_parser/
├── parser.py          # 主模块，对外暴露 AddressParser 类
├── trie.py            # Trie 树实现，用于地名精确匹配
├── corrector.py       # 错别字纠错模块
├── pinyin_index.py    # 拼音匹配模块（需 pypinyin）
├── es_backend.py      # Elasticsearch 后端（需 elasticsearch）
├── llm_backend.py     # LLM 大模型兜底后端（需 openai）
├── district_db.json   # 全国行政区划数据库（省/市/区三级）
└── city_alias.json    # 城市俗称/别名映射（可选）
```

调用方还需准备（放在项目目录下）：

```
简称映射.json          # 城市和区县的简称，如"海淀"→"海淀区"（可选）
```

## 快速开始

```python
from address_parser.parser import AddressParser

# 初始化（仅必填 district_db.json 路径）
parser = AddressParser(
    r"D:\your_project\address_parser\district_db.json"
)

# 解析地址
result = parser.parse("张三 13800138000 内蒙古呼和浩特市农大东区")
print(result)
# 输出：
# {
#   'name': '张三',
#   'phone': '13800138000',
#   'province': '内蒙古自治区',
#   'city': '呼和浩特市',
#   'district': None,
#   'detail_address': '农大东区'
# }
```

启用简称映射（推荐，覆盖城市/区县简称）：

```python
parser = AddressParser(
    db_path     = r"D:\your_project\address_parser\district_db.json",
    abbrev_path = r"D:\your_project\简称映射.json",
)
```

## AddressParser 初始化参数

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `db_path` | `str` | **必填** | `district_db.json` 文件路径 |
| `abbrev_path` | `str` | `None` | 简称映射 JSON 路径（如"海淀"→"海淀区"） |
| `alias_path` | `str` | `None` | 城市俗称/别名 JSON 路径（如"魔都"→"上海市"） |
| `enable_pinyin` | `bool` | `True` | 是否启用拼音匹配（需安装 `pypinyin`，未安装时自动跳过） |
| `enable_corrector` | `bool` | `True` | 是否启用错别字纠错（无额外依赖） |
| `enable_es` | `bool` | `False` | 是否启用 Elasticsearch 模糊搜索（需安装 `elasticsearch` 并启动 ES） |
| `enable_llm` | `bool` | `False` | 是否启用 LLM 大模型兜底（需安装 `openai` 并配置 API Key） |
| `es_hosts` | `list` | `None` | ES 节点地址，如 `["http://localhost:9200"]` |
| `llm_api_key` | `str` | `None` | LLM API Key（也可通过环境变量 `LLM_API_KEY` 设置） |
| `llm_base_url` | `str` | `None` | LLM 接口地址（默认 OpenAI 官方，可换 DeepSeek 等兼容地址） |
| `llm_model` | `str` | `None` | LLM 模型名称（默认 `gpt-4o-mini`） |

## parse() 返回字段说明

`parser.parse(text)` 接受任意字符串，返回一个字典：

| 字段 | 类型 | 说明 |
|---|---|---|
| `name` | `str` 或 `None` | 收件人姓名（位于文本最前，长度 ≤ 3 字） |
| `phone` | `str` 或 `None` | 手机号（11 位，1 开头，第二位 3-9） |
| `province` | `str` 或 `None` | 省份全称，如 `"广东省"`、`"内蒙古自治区"` |
| `city` | `str` 或 `None` | 城市全称，如 `"深圳市"`、`"呼和浩特市"` |
| `district` | `str` 或 `None` | 区/县全称，如 `"南山区"` ；无法识别时为 `None` |
| `detail_address` | `str` | 去掉省市区后的详细地址，如 `"科技园南区A座3楼"` |

**注意**：
- 姓名提取规则：第一个空格前的词，且长度 ≤ 3 字。若地址无空格分隔，或首词超过 3 字，则 `name` 为 `None`。
- `district` 在 `detail_address` 中无法找到合法区县时，返回 `None`，不会乱填。

## 解析流程与 Fallback 链

解析按以下优先级依次尝试，前一级成功则不会触发后续级别：

```
Level 1  Trie 精确匹配
         ↓ 未识别
Level 2  错别字纠错后重试 Trie（enable_corrector=True）
         ↓ 未识别
Level 3  拼音匹配（纯拼音输入，enable_pinyin=True）
         ↓ 未识别
Level 4  Elasticsearch 模糊搜索（enable_es=True）
         ↓ 未识别
Level 5  LLM 大模型兜底（enable_llm=True）
```

绝大多数中文地址在 Level 1 即可命中。Level 2~5 均为可选增强，按需开启。

## 简称映射配置

文件：`简称映射.json`，格式示例：

```json
{
  "直辖市": {
    "北京市": [["京", "北京"], {
      "海淀区": ["海淀"],
      "朝阳区": ["朝阳"]
    }]
  },
  "广东省": [["粤", "广东"], {
    "深圳市": [["深圳"], {
      "南山区": ["南山"],
      "福田区": ["福田"]
    }]
  }]
}
```

**格式说明**：

- 顶层 key 为省份全名（或 `"直辖市"` 分组）
- 省份对应值为 `[ [省份简称列表], { 城市字典 } ]`
- 城市字典：`城市全名 → [ [城市简称列表], { 区县字典 } ]`
- 区县字典：`区县全名 → [区县简称列表]`
- **省份简称不插入 Trie**（如"粤"、"苏"太短易误匹配），省份识别通过全名或 `complete_region` 反推实现
- 单字简称（长度 = 1）自动跳过，避免误匹配

## 城市别名配置

文件：`city_alias.json`（位于 `address_parser/` 目录下），格式：

```json
{
  "_注释": "城市俗称/别名映射，key 为别名，value 为行政区划全称",
  "魔都": "上海市",
  "帝都": "北京市",
  "蓉城": "成都市",
  "星城": "长沙市"
}
```

- 以 `_` 开头的 key 视为注释，自动跳过
- value 必须与 `district_db.json` 中的 `name` 字段完全一致，否则跳过并打印 debug 日志

启用方式：

```python
parser = AddressParser(
    db_path    = r"...\district_db.json",
    alias_path = r"...\address_parser\city_alias.json",
)
```

## 错别字纠错

默认开启（`enable_corrector=True`），无需额外安装依赖。

### 工作原理

纠错器作为 **Level 2 Fallback**，只在 Trie 精确匹配**完全失败**（省、市、区一个都没匹配上）时自动触发，对使用者透明：

```
Trie 匹配 → 省市区全部未命中
    ↓
对地址文本做错别字替换（先查词典，再做编辑距离匹配）
    ↓
用纠正后的文本重新做 Trie 匹配
```

若 Trie 已匹配到任意一个省/市/区，则不会进入纠错流程。

### 使用方式

纠错功能随 `parse()` 自动生效，无需额外调用：

```python
from address_parser.parser import AddressParser

parser = AddressParser(
    db_path = r"...\district_db.json",
    # enable_corrector=True  ← 默认值，不写也生效
)

# 含错别字的地址，自动纠正后解析
result = parser.parse("张三 13800138000 郑洲市金水区")
# "郑洲" 自动纠正为 "郑州市" → Trie 重新匹配成功
# → {'province': '河南省', 'city': '郑州市', 'district': '金水区', ...}

result = parser.parse("李四 13900000000 石家装市裕华区")
# "石家装" 自动纠正为 "石家庄市"
# → {'province': '河北省', 'city': '石家庄市', 'district': '裕华区', ...}
```

### 内置错别字词典

| 错误写法 | 纠正为 |
|---|---|
| `郑洲` | `郑州市` |
| `兰洲` | `兰州市` |
| `杭洲` / `杭洲市` | `杭州市` |
| `苏洲` / `苏洲市` | `苏州市` |
| `温洲` / `温洲市` | `温州市` |
| `福洲` | `福州市` |
| `广洲` / `广洲市` | `广州市` |
| `贵洲` / `贵洲省` | `贵州省` |
| `石家装` / `石家桩` / `石家庒` | `石家庄市` |
| `乌鲁木其` / `乌鲁木奇` | `乌鲁木齐市` |
| `哈滨` | `哈尔滨市` |
| `海甸区` / `海甸` | `海淀区` |
| `丰泰区` | `丰台区` |
| `浦东区` | `浦东新区` |

### 扩展自定义词典

在 `corrector.py` 的 `TYPO_DICT` 字典中追加条目，格式为 `错误写法: 正确地名全称`（全称须与 `district_db.json` 中的 `name` 字段完全一致）：

```python
# corrector.py
TYPO_DICT: Dict[str, str] = {
    # ... 已有条目 ...

    # 追加自定义条目
    "重庆直辖市": "重庆市",
    "大连市": "大连市",   # 保持不变也可（防误触发）
}
```

### 关闭纠错

```python
parser = AddressParser(
    db_path          = r"...\district_db.json",
    enable_corrector = False,   # 关闭后 Level 2 不再触发
)
```

### 可选加速

安装 `rapidfuzz` 可将编辑距离计算速度提升约 10 倍（词典替换本身无额外开销，编辑距离匹配在批量解析时效果明显）：

```bash
pip install rapidfuzz
```

未安装时自动降级为内置的动态规划实现，功能不受影响。

## 可选后端：Elasticsearch

适用场景：地址写法极不规范，Trie 精确匹配和纠错均无法命中时，用 ES 做模糊搜索兜底。

**Step 1：安装依赖**

```bash
pip install "elasticsearch>=8,<9"
```

**Step 2：启动 ES（Docker 方式）**

```bash
docker run -d --name es8 \
  -e "discovery.type=single-node" \
  -e "xpack.security.enabled=false" \
  -p 9200:9200 \
  docker.elastic.co/elasticsearch/elasticsearch:8.13.4
```

**Step 3：初始化索引（首次使用执行一次）**

```python
import json
from address_parser.es_backend import ESBackend

with open(r"...\district_db.json", encoding="utf-8") as f:
    db = json.load(f)

backend = ESBackend()            # 默认连接 http://localhost:9200
backend.build_index(db)          # 将全量行政区划数据写入 ES
```

**Step 4：启用 ES 后端**

```python
parser = AddressParser(
    db_path   = r"...\district_db.json",
    enable_es = True,
    es_hosts  = ["http://localhost:9200"],  # 默认值，可省略
)
```

## 可选后端：LLM 大模型兜底

适用场景：地址极度不规范或包含方言描述，所有规则化方法均失败时，调用大模型进行语义理解。

**支持的服务**：OpenAI、DeepSeek、Moonshot、通义千问、本地 Ollama 等 OpenAI 兼容接口。

**Step 1：安装依赖**

```bash
pip install openai
```

**Step 2：配置 API Key（推荐用环境变量）**

```bash
# PowerShell
$env:LLM_API_KEY = "sk-xxxxxxxx"
$env:LLM_BASE_URL = "https://api.deepseek.com/v1"   # 使用 DeepSeek 时
$env:LLM_MODEL = "deepseek-chat"
```

**Step 3：启用 LLM 后端**

```python
# 方式一：读取环境变量（推荐）
parser = AddressParser(
    db_path    = r"...\district_db.json",
    enable_llm = True,
)

# 方式二：直接传参
parser = AddressParser(
    db_path      = r"...\district_db.json",
    enable_llm   = True,
    llm_api_key  = "sk-xxxxxxxx",
    llm_base_url = "https://api.deepseek.com/v1",
    llm_model    = "deepseek-chat",
)
```

**费用提示**：LLM 兜底每次调用会产生 API 费用，建议仅在其他级别均失败时触发（默认行为即如此，无需额外配置）。

## 常见问题

**Q：姓名被错误提取了怎么办？**

当前规则：文本开头第一个空格前的词，长度 ≤ 3 字时视为姓名。若地址首段词恰好是 1-3 字且不含空格，会被当作姓名。
解决方法：输入时确保姓名和地址之间有空格，或在姓名后加空格分隔。

---

**Q：`district` 始终为 `None`，但地址里明明有区名？**

可能原因：
1. 该区名不在 `district_db.json` 数据库中（如某些开发区、新区）
2. 区名与城市不匹配（如地址写了错误的城市），被结构校验清空
3. 区名长度 ≤ 2 字，被过滤（防误匹配），可通过完整区名（如"南山区"而非"南山"）解决

---

**Q：`detail_address` 中出现了省市区的名字残留？**

可能原因：地址中省市区名称写法与数据库不完全一致，导致 `extract_detail` 未能正确删除。
解决方法：启用简称映射（`abbrev_path`）或补充到错别字词典（`TYPO_DICT`）。

---

**Q：Python 3.6 能否使用？**

可以。代码中所有类型注解均使用 `typing` 模块的写法（如 `Optional[str]`、`List[dict]`），兼容 Python 3.6+。

---

**Q：如何批量解析？**

```python
addresses = [
    "李四 13900139000 广东省深圳市南山区科技园南路",
    "王五 18800000000 北京海淀中关村大街1号",
    "赵六 13700137000 上海浦东新区张江高科技园",
]

results = [parser.parse(addr) for addr in addresses]
for r in results:
    print(r)
```

---

**Q：`pypinyin` 未安装时会报错吗？**

不会报错，`enable_pinyin=True` 时若检测到 `pypinyin` 未安装，会打印一条警告日志并自动跳过拼音模块，不影响其他解析功能。ES、LLM 后端同理。
