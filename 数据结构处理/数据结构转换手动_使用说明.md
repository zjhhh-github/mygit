# 数据结构转换手动.py 使用说明

## 一、脚本功能概述

本脚本用于将报名录入 Excel 表格中的学员数据，整合微信通讯录数据库、地址解析、宝妈身份判断、公司归属等多个来源，最终转换为结构化的 JSON 文件，供后续系统导入或数据分析使用。

---

## 二、依赖环境

### Python 包
| 包名 | 用途 |
|------|------|
| `pandas` | 读取 Excel 文件、处理表格数据 |
| `numpy` | 数据类型转换辅助 |
| `json` | 读写 JSON 文件 |
| `sqlite3` | 查询微信通讯录数据库 |
| `pathlib` | 路径处理 |
| `address_parser`（本地模块） | 将收件人地址解析为省/市/区/详情 |

### 安装命令
```bash
pip install pandas numpy openpyxl
```

> `address_parser` 为本地自定义模块，位于脚本同目录下的 `address_parser/` 文件夹中，无需单独安装。

## 四、输入数据说明

### 报名录入.xlsx（三个工作表，均从第2行起为表头）

**① 工作表：报名录入**

主数据表，每行为一条报名记录，脚本读取以下列：

| 列名 | 说明 |
|------|------|
| `编号` | 学员唯一 ID |
| `编号+孩子中文全名` | 用于匹配宝妈身份和微信备注的关键字 |
| `孩子中文全名` | 孩子姓名 |
| `收件人电话` | 手机号 |
| `收件人地址` | 快递收件地址（用于解析省市区） |
| `推荐` | 推荐人信息（取"-"前部分） |
| `交易订单编号` | 报名订单号 |
| `报名项` | 报名课程项目 |
| `订单金额` | 支付金额 |
| `交易时间` | 报名时间 |
| `单号` | 快递单号 |
| `激活码` | 课程激活码 |
| `校区` | 所在校区 |
| `老师` | 对应老师 |

**② 工作表：内部通讯录**

用于补充每位学员的渠道和带领信息，读取以下列：

| 列名 | 说明 |
|------|------|
| `学员` | 与报名录入中"编号+孩子中文全名"对应 |
| `渠道C` | 渠道信息（取"-"前部分） |
| `带领C` | 带领人信息（取"-"前部分） |

**③ 工作表：宝妈**

用于判断学员是否为合伙宝妈，读取以下列：

| 列名 | 说明 |
|------|------|
| `宝妈` | 宝妈名单（与"编号+孩子中文全名"匹配） |
| `分账编号` | 对应的慧分账编号 |

## 五、处理流程说明

```
读取 Excel（3个工作表）
        ↓
构建辅助查找字典
  - bm2hfz：宝妈 → 慧分账编号
  - xueyuan2qddl：学员 → [渠道, 带领]
  - zuzhang2company：组长 → 公司名称
        ↓
遍历每条报名记录，逐条处理：
  1. 判断是否为合伙宝妈（✅/❌）
  2. 查询 SQLite 数据库获取微信信息
     - 查不到则标记"未添加好友"
  3. 解析收件地址 → 省/市/区/县/详细地址
     - 解析失败则所有地址字段置空
  4. 从辅助字典中查找渠道、带领、公司名称、慧分账编号
  5. 组装为结构化 JSON 记录
        ↓
对 result 列表按 ID 去重
  - ID 相同时，保留信息量更多的那条
  - 信息量 = 递归统计所有非空字段数量
        ↓
写入输出 JSON 文件
```

## 七、ID 去重逻辑说明

当 Excel 中存在同一编号（`ID`）对应多行数据时，脚本会自动去重，规则如下：

- 对每条记录递归统计所有非空字段数量，得到"信息得分"
- 同一 ID 的多条记录中，**保留得分最高的一条**
- 若得分相同，保留在 Excel 中**先出现**的那条
- 运行时若有重复，控制台会输出提示：
  ```
  [去重] 共发现 X 条重复ID记录，已保留信息更多的一条。
  ```

以下值在计分时视为"空"，不计入得分：

> 空字符串 `""`、`None`、`"⚠️"`、`"❌"`、`"未添加好友"`、`"原始地址为空"`

## 八、常见问题

**Q：运行报错 `连接数据库失败`**
> 检查 `contact.db` 文件是否存在于 `C:\Users\LENOVO\Desktop\` 目录下，以及文件是否被其他程序占用。

**Q：某条记录地址字段全部为空**
> 该记录的收件人地址无法被解析（格式异常或地址过短），属于正常降级处理，不影响其他字段。

**Q：`慧分账编号` 显示 `⚠️`**
> 该学员的"编号+孩子中文全名"未出现在"宝妈"工作表中，需手动核查。

**Q：`公司名称` 显示 `❌`**
> 该学员未出现在 `小组结构图.json` 的组长名单中，无对应公司信息。

**Q：`微信原始ID` 显示 `未添加好友`**
> SQLite 数据库中未找到以该学员名为备注的联系人，说明微信尚未添加或备注不一致。
