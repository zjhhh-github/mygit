# 上传用户结构 - 项目介绍

## 1. 项目定位

`上传用户结构.py` 是一个**单文件数据转换脚本**，用于把 `_脚本输入_1.txt` 中的原始行数据，转换为“可直接用于上传”的标准 JSON 用户结构。

它的目标是：在不做复杂业务推断的前提下，稳定、可回滚地完成字段提取与格式统一。

---

## 2. 解决的问题

在实际数据处理中，原始文本通常存在以下问题：

- 行格式不统一，包含空行、说明行、非目标数据行；
- 同一账号可能重复出现，导致上传冲突；
- 下游上传系统需要固定字段名称与固定角色值。

本脚本通过“过滤 + 解析 + 去重 + 结构化输出”解决上述问题。

---

## 3. 输入与输出

### 输入

- 文件路径：`C:\Users\LENOVO\Desktop\_脚本输入_1.txt`
- 输入特征：文本行中包含 `¿¿¿` 标识，目标字段位于制表符分列中。

### 输出

- 文件路径：`D:\桌面文件\新建文件夹\数据结构处理\意向学员绑定关系查询系统\上传用户结构.json`
- 输出格式：JSON 数组，每个元素是一个用户对象。

---

## 4. 处理流程

脚本主流程如下：

1. 读取输入文本全部行；
2. 对每一行做有效性筛选（是否包含 `¿¿¿`）；
3. 按制表符拆分字段并提取：
   - `账号`：第 2 列（代码中的 `列表[1]`）
   - `密码`：第 3 列（`列表[2]`）
   - `微信号`：第 4 列（`列表[3]`）
4. 账号去重（同账号只保留首次出现）；
5. 补充固定字段 `角色 = "普通用户"`；
6. 写入目标 JSON 文件。

---

## 5. 输出数据结构

单条输出示例：

```json
{
  "账号": "示例账号",
  "密码": "示例密码",
  "微信号": "示例微信号",
  "角色": "普通用户"
}
```

---

## 6. 核心规则（当前版本）

- **规则1：仅处理节点行**
  - 行中不含 `¿¿¿` 的内容直接跳过。
- **规则2：列数不足跳过**
  - 拆分后少于 4 列不入库，防止输出脏结构。
- **规则3：账号为空跳过**
  - 保证上传对象至少有唯一主键。
- **规则4：账号去重**
  - 同账号重复时仅保留第一条，避免上传冲突。
- **规则5：角色固定**
  - 所有用户统一写入 `"普通用户"`，降低下游复杂度。

---

## 8. 稳定性与风险提示

- **输入格式强依赖制表符列位**  
  若源数据不是以制表符分列，或列顺序变化，解析结果会偏差。

- **去重策略为“首次保留”**  
  若后出现的数据更准确，当前策略不会自动覆盖。

- **仅做结构转换，不做业务校验**  
  例如密码强度、微信号格式、账号合法性等未做校验。

- **固定路径运行**  
  跨机器使用时需调整脚本内输入输出路径。

---

- 在上传前快速做账号级去重；
- 为后续接口上传或后台导入提供标准 JSON。
