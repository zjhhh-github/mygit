# 散件发货回填脚本说明

## 1. 脚本作用

`read_feishu_bitable.py` 用于读取飞书多维表格数据，并按“散件名称 -> 编号/金额”的映射关系，自动回填目标表中的以下字段：

- `散件编号(最终)`
- `散件总金额`

同时，脚本会基于回填结果自动新增“拆分行”（并写入拆分标记），用于后续发货处理。

## 3. 处理规则（核心）

### 3.1 匹配与金额计算

1. 按“完全匹配”查找映射表中的散件名称（不做模糊匹配）。
2. 数量字段名由散件名称推导：把最后一个 `【...】` 替换为 `【数量】`，例如：
   - `牛津树 绘本（套装）【300元】` -> `牛津树 绘本（套装）【数量】`
3. 金额优先读取映射表 `金额(对外展示)`（或金额/价格），若解析不到数字则回退从名称中的 `【xx元】` 提取。
4. 单个散件命中多条映射时：
   - 编号会合并去重后继续处理
   - 金额会按所有有效单价累计

### 3.2 编号回填规则

- 每个匹配到的编号都按 `编号*数量` 格式输出。
- 多个编号用英文逗号拼接后写入 `散件编号(最终)`。

### 3.3 拆分新增规则

脚本会在更新原记录后新增拆分行，且会跳过已标记“拆分”的记录，避免重复执行导致指数级新增。

- 强制单独拆分项：`西游记绘本套装【100元】`
- 映射表中“同名重复”的散件会按编号逐条拆分，并把金额均摊到每条
- 其他散件默认合并为一条拆分行
- 指定散件名称会进一步把 `*N` 展开成 N 条 `*1`（并均摊金额）

### 3.4 拆分标记字段

- 默认使用字段名：`文本5`
- 若字段名有空格差异（如 `文本 5`），脚本会尝试自动识别
- 新增拆分行时，会把该字段写为：`拆分`

## 4. 运行前准备

1. 在飞书开放平台获取应用凭证：
   - `App ID`
   - `App Secret`
2. 确认应用具备多维表格读写权限（读取记录、更新记录、新增记录）。
3. 确认目标表至少包含以下字段：
   - `散件名称`
   - `散件编号(最终)`
   - `散件总金额`
   - 拆分标记字段（默认 `文本5`）
4. 确认映射表至少包含：
   - 名称字段（如 `散件名称`/`物品名称`/`商品名称`/`名称`）
   - `编号(对外展示)`
   - 金额字段（建议 `金额(对外展示)`）

## 5. 命令参数

| 参数 | 是否必填 | 说明 |
| --- | --- | --- |
| `--app-id` | 否 | 飞书应用 App ID，默认使用脚本内置值 |
| `--app-secret` | 否 | 飞书应用 App Secret，默认使用脚本内置值 |
| `--app-token` | 否 | 多维表格 App Token，默认使用脚本内置值 |
| `--target-table-id` | 否 | 回填目标表 Table ID |
| `--mapping-table-id` | 否 | 映射表 Table ID |
| `--target-view-id` | 否 | 目标表 View ID，传空可不按视图筛选 |
| `--mapping-view-id` | 否 | 映射表 View ID，传空可不按视图筛选 |
| `--page-size` | 否 | 分页大小，默认 100，范围 1~500 |
| `--dry-run` | 否 | 仅预览，不写回、不新增 |
| `--print-debug` | 否 | 打印读取与待写入明细，便于核对 |
| `--split-flag-field` | 否 | 拆分标记字段名，默认 `文本5` |

## 6. 执行方式

以下命令都建议在脚本目录执行：

- 相对路径：`小工具/散件发货回填`

### 6.1 Windows（PowerShell）

```powershell
cd D:\git\mygit\小工具\散件发货回填
python .\read_feishu_bitable.py --dry-run --print-debug
```

### 6.2 macOS / Linux（bash）

```bash
cd /path/to/mygit/小工具/散件发货回填
python3 ./read_feishu_bitable.py --dry-run --print-debug
```

### 6.3 确认无误后正式写回

把 `--dry-run` 去掉即可：

```powershell
python .\read_feishu_bitable.py
```

如需覆盖默认凭证，可显式传参：

```powershell
python .\read_feishu_bitable.py --app-id "cli_xxx" --app-secret "xxx"
```

## 7. 输出结果说明

执行结束会打印汇总信息，例如：

- 总处理条数
- 成功条数
- 跳过条数（无 record_id 或已是拆分行）
- 警告条数（如名称未匹配、数量无效、金额无效）
- 计划拆分条数 / 实际拆分条数

此外还会输出：

- `[警告][record_id] ...`：数据异常或匹配异常
- `[预览][record_id] ...`：`--dry-run` 模式下的回填预览
- `[拆分预览][record_id] ...`：`--dry-run` 模式下的拆分新增预览

## 8. 常见问题

### 8.1 执行失败：`--page-size 必须在 1~500 之间`

将 `--page-size` 调整到合法范围内（建议 100）。

### 8.2 报“读取多维表格失败”或“获取 tenant_access_token 失败”

优先检查：

1. `app-id` / `app-secret` 是否正确
2. 飞书应用权限是否开通
3. App 是否有目标多维表读写权限

### 8.3 出现大量“映射表散件名称未找到完全匹配项”

说明目标表散件名称与映射表名称存在文本不一致。该脚本是“完全匹配”策略，需先清洗名称数据再执行。

---

## 9. 最小验证步骤（推荐）

1. 先执行 `--dry-run --print-debug`，确认：
   - `散件编号(最终)` 格式正确（`编号*数量`）
   - `散件总金额` 符合预期
   - 拆分预览条数和内容合理
2. 去掉 `--dry-run` 正式执行。
3. 回到飞书表格核验：
   - 原记录是否已回填两个字段
   - 是否新增了拆分行
   - 新增行是否写入拆分标记（默认 `文本5=拆分`）
