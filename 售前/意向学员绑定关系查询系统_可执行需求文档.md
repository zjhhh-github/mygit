# 意向学员绑定关系查询系统（方案A）可执行需求文档

## 1. 结论与实施基线

- 本期采用**单体三层架构**：`Controller -> Service -> Repository`。
- 所有“解绑日期、绑定状态、来源优先级”逻辑统一收敛到**绑定规则服务**，避免前后台规则不一致。
- 采用“**最小改动、可回滚**”原则：先建基础能力，再按页面分批交付。

---

## 2. 目标与范围

### 2.1 产品目标

构建一套面向教育机构的意向学员绑定关系管理平台，支持：

1. 普通用户按意向学员微信号查询绑定关系并在允许条件下提交“添加来源申请”；
2. 管理员管理用户、意向学员、来源记录、申请审核和默认绑定周期；
3. 支持 JSON 导入导出与批量操作。

### 2.2 本期实现范围

- 登录鉴权（账号密码登录）
- 前端查询页 + 查询结果页 + 申请入口
- 后台用户管理
- 后台意向学员管理（含来源记录、申请记录）
- 绑定周期管理（系统默认值）
- JSON 导入导出、批量删除

### 2.3 本期不实现

- 用户注册
- 找回密码
- 审核通知推送（短信/微信）
- 数据统计报表
- 操作日志
- 多条“有绑定”并存时的高级排序策略（按本文默认策略）

---

## 3. 角色与权限

### 3.1 角色定义

- 普通用户：仅可访问前端查询能力（查询 + 提交申请）
- 管理员：可访问后台全部能力

### 3.2 权限控制规则

- 未登录用户访问受保护页面，统一跳转登录页
- 普通用户访问后台页面，返回无权限并跳转登录页（或无权限页）
- 管理员可访问全部管理接口

---

## 4. 数据模型（建议落地）

> 说明：字段命名可按实际项目风格调整，但语义必须一致。

### 4.1 用户表 `users`

- `id`：主键
- `account`：账号，唯一索引
- `password_hash`：密码哈希（禁止明文存储）
- `role`：枚举（`admin` / `user`）
- `created_at` / `updated_at`

### 4.2 意向学员表 `prospects`

- `id`：主键
- `wechat_id`：意向学员微信号，唯一索引
- `created_at` / `updated_at`

### 4.3 来源记录表 `prospect_sources`

- `id`：主键
- `prospect_id`：外键 -> `prospects.id`，索引
- `source_wechat_id`：来源微信号
- `bind_date`：绑定日期（`yyyyMMdd`）
- `bind_period_days`：绑定周期（可为空，为空时使用系统默认绑定周期）
- `created_by`：操作人（可选）
- `created_at` / `updated_at`

> 说明：`unbind_date` 与 `bind_status`为**计算字段**，不落库。

### 4.4 添加来源申请表 `source_applications`

- `id`：主键
- `prospect_id`：外键 -> `prospects.id`，索引
- `source_wechat_id`：申请来源微信号
- `apply_date`：申请日期（作为审核通过后的绑定日期）
- `bind_period_days`：申请时绑定周期（可为空）
- `prospect_wechat_screenshot_url`：意向学员微信号截图（1张）
- `chat_screenshots_json`：聊天截图列表（1~9张）
- `status`：枚举（`pending` / `approved` / `rejected`）
- `reviewed_by` / `reviewed_at` / `review_remark`
- `created_at` / `updated_at`

### 4.5 系统配置表 `system_configs`

- `config_key`：主键（如 `default_bind_period_days`）
- `config_value`
- `updated_at`

## 5. 核心业务规则（统一由绑定规则服务实现）

### 5.1 解绑日期计算

- 公式：`解绑日期 = 绑定日期 + 绑定周期(天)`
- 示例：`20260320 + 10` -> `20260330`

### 5.2 绑定状态计算

- 当 `解绑日期 < 当天` -> `无绑定`
- 当 `解绑日期 >= 当天` -> `有绑定`

### 5.3 绑定周期优先级

1. 来源记录有单独设置 -> 用来源记录周期
2. 来源记录未设置 -> 用系统默认周期
3. 默认周期未设置且需要计算 -> 返回业务错误

### 5.4 来源显示优先级（前端查询与后台列表统一）

1. 无来源 -> 来源字段全空
2. 有来源且存在“有绑定” -> 取“绑定日期最新”的那条
3. 无“有绑定” -> 取“解绑日期最近（最大）”的那条

### 5.5 申请入口显示条件

- 仅当当前展示记录绑定状态为空或“无绑定”时展示
- 绑定状态为“有绑定”时不展示

### 5.6 后台手动新增来源限制

- 目标意向学员当前状态为“有绑定”时，禁止新增来源记录

### 5.7 申请审核规则

- 通过：写入来源记录（`source_wechat_id + apply_date + bind_period_days`）
- 拒绝：仅更新申请状态，不写来源记录

## 6. 页面与接口清单（可直接进入开发）

## 6.1 登录

- `POST /api/auth/login`
  - 入参：`account`、`password`
  - 出参：`token`、`role`
  - 异常：统一提示“账号或密码错误”

## 6.2 前端查询

- `GET /api/query/prospect?wechatId=xxx`
  - 返回：意向学员微信号 + 当前展示来源5字段 + 是否可申请
  - 不存在：提示“未找到该意向学员信息”

- `POST /api/query/source-application`
  - 入参：`prospectWechatId`、`sourceWechatId`、`prospectScreenshot`、`chatScreenshots[]`
  - 校验：截图1张必填、聊天截图1~9张
  - 结果：进入待审核

## 6.3 后台用户管理（管理员）

- `GET /api/admin/users`（按账号/角色筛选）
- `POST /api/admin/users`（单条新增）
- `PUT /api/admin/users/{id}`（编辑账号/密码/角色）
- `DELETE /api/admin/users/{id}`（单删）
- `POST /api/admin/users/batch-delete`（批删）
- `POST /api/admin/users/import-json`（JSON导入）
- `GET /api/admin/users/export-json`（JSON导出）

## 6.4 后台意向学员管理（管理员）

- `GET /api/admin/prospects`（按微信号筛选，返回列表8字段）
- `POST /api/admin/prospects`（单条新增）
- `PUT /api/admin/prospects/{id}`（编辑）
- `DELETE /api/admin/prospects/{id}`（单删）
- `POST /api/admin/prospects/batch-delete`（批删）
- `POST /api/admin/prospects/import-json`（JSON导入）
- `GET /api/admin/prospects/export-json`（JSON导出）

### 6.4.1 来源记录子接口

- `GET /api/admin/prospects/{id}/sources`（展开查看所有来源）
- `POST /api/admin/prospects/{id}/sources`（手动添加来源，受“有绑定禁止新增”限制）

### 6.4.2 申请记录子接口

- `GET /api/admin/prospects/{id}/applications`（展开查看申请）
- `GET /api/admin/applications/{id}`（查看详情含截图）
- `POST /api/admin/applications/{id}/approve`（审核通过）
- `POST /api/admin/applications/{id}/reject`（审核拒绝）

## 6.5 绑定周期管理

- `GET /api/admin/bind-period/default`
- `PUT /api/admin/bind-period/default`
  - 入参：`defaultBindPeriodDays`（正整数）

## 7. 导入导出规则

### 7.1 用户导入导出

- 导入字段：账号、密码、角色
- 导出字段：账号、密码、角色（如涉及安全要求，建议导出脱敏或仅管理员可用）

### 7.2 意向学员导入

- 支持结构：意向学员微信号 + 来源数组
- 每条来源允许不传 `绑定周期`
- `解绑日期` 与 `绑定状态`可不传，按规则实时计算
- 当来源缺失绑定周期且系统默认绑定周期未设置：整批失败并提示

### 7.3 意向学员导出

- 导出完整字段，并包含计算后的 `解绑日期`、`绑定状态`

### 7.4 事务与失败策略

- 导入采用“先校验后写入”策略
- 任一条格式错误：终止导入，不影响原有数据

---

## 8. 校验规则

- 微信号字段：非空、长度上限（建议 64）
- 绑定日期：必须是合法 `yyyyMMdd`
- 绑定周期：正整数，建议上限 3650
- 聊天截图：1~9 张
- 申请提交时：必须确认当前展示状态为空或“无绑定”

---

## 9. 并发与一致性控制

- 审核通过接口需做幂等：同一申请仅允许处理一次
- 审核通过写入来源记录时建议事务化处理：
  1. 锁定申请记录（或版本号校验）
  2. 校验状态仍为 `pending`
  3. 写入来源记录
  4. 更新申请为 `approved`

---

## 10. 验收用例映射（最小集）

1. 用初始管理员账号登录成功并进入后台
2. 普通用户登录后仅可访问前台查询
3. 查询结果来源显示符合优先级规则
4. 状态为空/无绑定显示申请入口；有绑定不显示
5. 申请表单校验生效（截图必填、聊天截图<=9）
6. 管理员通过/拒绝审核逻辑正确
7. 解绑日期与绑定状态计算准确
8. JSON 导入导出格式与规则正确
9. 批量删除、批量导出可用
10. 默认绑定周期设置后对未单独设置记录生效
11. 后台来源记录/申请记录可展开查看
12. 当前有绑定时后台禁止新增来源

## 12. 风险提示（工程视角）

- 若导出“密码明文”会有安全风险，建议改为仅导出账号和角色，或导出脱敏密码占位。
- 绑定状态实时计算在数据量大时会有性能压力，建议为 `prospect_id`、`bind_date`建立索引并分页查询。
- 图片存储建议统一对象存储并设置访问权限，避免敏感数据泄露。
